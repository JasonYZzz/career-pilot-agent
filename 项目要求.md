# AI Agent 机试题

## 任务背景

我们希望了解你如何设计一个小型但完整的 AI Agent。这个 Agent 不需要做 UI、部署或复杂工程架构,重点是看你如何组织模型上下文、接入工具和 skill、控制 token 预算,并处理真实运行中的边界情况。

你可以使用任意编程语言、模型 API、第三方库和 AI 编程工具。本题不考察手写代码熟练度,更关注你在 Agent 架构上的品味。

## 任务要求

请从 0 实现一个命令行或本地服务形式的 AI Agent。它需要能接收用户任务,自主决定是否调用工具或 skill,在有限 token 预算内完成多步任务,并导出可复盘的运行 trace。

需要支持以下能力:

1. **Agent 核心循环**  
   支持多轮推理和多步任务推进,能在继续调用工具和给出最终答案之间做判断,并有明确的终止条件。

2. **Tool 能力**  
   至少支持 3 个工具。建议从文件读取、文件写入、目录检索、todo 状态管理、时间获取、提醒创建、受限 shell 执行中选择。  
   如果实现 shell 执行,需要有工作目录限制、超时、错误处理和高风险命令防护。

3. **Skill 能力**  
   支持从本地目录加载 skill。一个 skill 可以是一份 markdown 指令文档,也可以带配置或脚本。Agent 应能按任务需要渐进式加载相关 skill,而不是启动时把所有 skill 全量放进上下文。

4. **Token 预算与压缩**  
   需要有每轮 token 预算意识。可以使用近似估算,但要有预算上限、触发压缩的水位和压缩策略。压缩后,任务目标、关键约束、当前进度、未完成事项和重要工具结果应尽量保留。

5. **边界处理**  
   请考虑工具失败、工具超时、工具结果过大、prompt injection、隐私数据访问、长期记忆写入、未经确认的提醒创建、高风险 shell 命令、重复失败动作、预算不足等情况。

6. **可观测 Trace**  
   每次运行应能导出 trace,用于复盘 Agent 的决策过程。trace 至少包含模型调用、工具调用、skill 加载、token 估算、压缩触发、边界处理和耗时信息。

## 交付物

请提交:

1. 代码仓库或压缩包。
2. README,包含安装方式、运行命令、模型配置方式和一个示例任务。
3. 至少 1 份完整 trace 样例,其中应包含工具调用和上下文压缩。
4. 简短说明你如何使用 AI 编程工具,以及你在架构和关键取舍上的判断。

建议提供类似下面的运行入口,便于我们复现:

```bash
agent run --task "读取 data 目录下的文档并生成报告" --workspace ./workspace --trace ./trace.json
```

如果入口不同,请在 README 中提供等价的一条运行命令。
